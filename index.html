What this version does

This is the Bristol-focused front end. It scores listings, ranks them, explains why they matter, and is structured so a future scraper or feed can drop straight into the data model.

  • Bristol-area filters and ranked acquisition cards
  • Investment rationale written for a decision-maker
  • Scoring based on discount, £/sqft, refurb upside and yield
  • Ready for automatic listing ingestion via a backend endpoint

Automatic scanning path

Client-side scraping is unreliable and often blocked. The correct build is: source connector → normaliser → score engine → ranked API → this front end. This page is already wired for that shape.
